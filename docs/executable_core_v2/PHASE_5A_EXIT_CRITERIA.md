# Phase 5A exit criteria

Phase 5A closes only when all of the following hold at one exact implementation head:

- [ ] Phase 0 contract validation passes.
- [ ] Phase 1 through Phase 4 regression suites pass.
- [ ] Python source quality reports no placeholder or forbidden backend issue.
- [ ] Linux, Windows, and macOS execute the separate-process generator suite.
- [ ] The worker input schema rejects checker source, trust anchors, previous manifests, promotion ledgers, and reference answers.
- [ ] The proposal schema contains no certificate, candidate, successor, or acceptance field.
- [ ] The worker source guard rejects direct file, process, network, privileged-import, and dynamic module-table capabilities.
- [ ] The worker runs in a separate isolated process with no file arguments, network endpoints, or write handles.
- [ ] The `initial` predecessor deterministically emits only the bounded `improve` word.
- [ ] The `target` predecessor deterministically emits only the bounded `stabilize` word.
- [ ] Word depth, proof length, proposal count, and resource bounds match the Lean seed library.
- [ ] Two fresh worker executions produce identical bytes, proposals, and process reports.
- [ ] Certificate construction is performed outside the worker.
- [ ] Selection is deterministic and consumes no model score or reference answer.
- [ ] The successor is derived by applying the selected update; no generator successor field or manual successor output is consumed.
- [ ] The generator predecessor-manifest hash matches the independently reconstructed checker predecessor manifest.
- [ ] The pinned Formal Core project builds.
- [ ] Generated Lean source passes the pre-compilation `sorry`/`sorryAx`/`admit`/local-axiom gate.
- [ ] Both reference transitions receive accepting pinned Lean reports.
- [ ] Both reference transitions receive accepting Phase 4 hardened checker reports.
- [ ] Every input, output, stage result, Lean report, checker report, and final report hash is retained.
- [ ] The final workflow closure artifact identifies the exact checked head.

The following remain false after Phase 5A:

```text
generator trust
candidate promotion authorization
real filesystem/package realization
selector ranking beyond the single bounded proposal
open-ended search or program synthesis
LLM or scaffold proposal generation
PyTorch proposal or training-driven updates
independent replay without the original generator
external benchmark claims
general noncommuting quantum semantics
autonomous or unbounded RSI
```

A clean Phase 5A closure licenses Phase 5B untrusted-generator experiments and Phase 6
realizer/package-builder development. Neither path enters the promotion trust boundary.
